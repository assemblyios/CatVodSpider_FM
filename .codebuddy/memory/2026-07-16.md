## CatVodSpider 项目认知
- 项目本质：CatVodTVSpider 的派生版，是一个「爬虫(站点解析)开发与调试沙盒」，产物为 `custom_spider.jar` 供 CatVod 系播放器加载。
- 核心：`app/src/main/java/com/github/catvod/spider/` 下各站点解析器继承 `crawler/Spider.java`；`MainActivity` 硬编码 `spider = new PTT();` 作为调试目标；`json/config.json` 用 `csp_类名` 关联 spider。
- 调试：Android Studio 直接 Run app，界面按钮对应 spider 各方法，返回 JSON 显示在屏幕 + Logcat。
- 打包：`build.bat`(Win) → gradlew assembleRelease → apktool 抽 smali → 生成 custom_spider.jar → checkJar.ps1 校验。
- macOS 环境：.bat 不能直接跑，需用 ./gradlew + apktool(brew) + pwsh 替代。

## 关键认知：两套 CatVod 爬虫体系不互通
- jar 派（本项目）：Java 写 Spider → custom_spider.jar → FongMi/TV 用 DexClassLoader + csp_ 反射加载。
- JS 派（jadehh/TVSpider）：JS(QuickJS/Node) 写 Spider → 发布到 git 分支(js/dist) → TVBox/CatVodOpen 用 `gitee://Token@...` 导入配置串加载。
- 两者实现语言、产物形态、目标播放器、分发机制均不同，不互通。

## XBPQ 规则源（用户问的「xqbq源」实为 XBPQ 源）
- 本质：TVBox/CatVod 生态的「规则驱动型爬虫解析器」，约300KB的 `xbpq.jar`（自定义 csp spider, type=3），内置规则引擎。
- 三种写源方式第3种：jar源(本项目,Java高成本) / js源(jadehh,JS中成本) / XBPQ规则源(只写JSON规则,低成本)。
- 原理：配置 `"api":"csp_XBPQ"`，`ext` 里用 `&&`/`$$`/`*`/`[替换]`/`[不包含]`/JSON路径/Base64 等截取符号，按 数组/线路/播放/搜索 四块抽数据。
- 与本项目关系：XBPQ 本身是个第三方 spider jar，加载机制与本项目一致(DexClassLoader+csp_)，可「精缝」合并进本项目工程一起打 custom_spider.jar。已补入 docs/开发文档.md 第十节。
- 用户遇到的「XBPQ源」通常指别人维护的基于 xbpq.jar 的配置仓库，如 boyss180/gao-xBPQ（xBPQ.json，jar与配置来自「菜妮丝」）。

## 反编译 xBPQ.jar 实战记录（2026-07-16）
- 工具：macOS `brew install jadx`(1.5.6)；`curl` 下载；jar 内是 classes.dex。
- 样本：`https://raw.githubusercontent.com/boyss180/gao-xBPQ/master/jar/xBPQ1107.jar`（354KB，构建于2022-11-07，菜妮丝线）。反编译产物在 /tmp/xbpq_analysis（工作区外，无需清理）。
- **xbpq.jar 无公开 Java 源码仓库**，属黑盒引擎，仅以 .jar 分发。主类 `com.github.catvod.spider.xBPQ extends Spider`，重写 homeContent/homeVideoContent/categoryContent/playerContent。
- **字符串混淆**：`merge.BfT.d(hex)` = hex解码后逐字节 XOR 密钥 `"jDLKJK"`。可据此解出内部符号：`&`(4C)、`#`(49)、`>>`(547A)、`[替换`(31A2D7F4ACC6C8)、`[追加`(31ACF3F6AFC1CA)、`json数组`、Base64(android.util.Base64)等。
- **真实规则语法（与教程略有出入，以 LIBVIO.json 为准）**：`&`列表分隔、`&&`边界截取、`{cateId}{catePg}{area}{by}{year}{class}`模板、`[不包含:a#b]`(#分隔多关键词)、`[替换:原>>新]`、`[追加:..]`、Base64()、json数组 JSON路径模式。
- 已产出文档 `docs/XBPQ规则引擎分析.md`（反编译步骤/包结构/混淆解密/语法/核心逻辑/与本项目集成）。
- 新增 `docs/XBPQ站点选型指南.md`：判断网站适不适合用 XBPQ 规则写。核心=四道硬门槛(源码可见/URL可模板化/直链可截/无强反爬)+三类站(maccms强适合/JSON接口勉强/SPA-JS不适合)+5分钟自测+体检清单。口诀：「源码可见、地址有律、直链好取、没有强盾」→XBPQ；「渲染靠JS、参数靠算、播放靠解密」→写jar/JS。
- 新增 `docs/CatVod源接入实战教程.md`：端到端总教程。结构=0.决策树(选路线A-XBPQ规则 / 路线B-写爬虫)+路线A(选型→写规则四符号&字段步骤&接入播放器&排错顺序)+路线B(不适合XBPQ时按目标播放器选Java[jar派/FongMi TV,见开发文档]或JS[JS派/TVBox-CatVodOpen,见jadehh文档])+速查表+体检清单+文档索引。用户意图：系统学习"怎么把网站变成可播放源"。
