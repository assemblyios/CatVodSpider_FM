# CatVodSpider 开发文档

## 一、项目是干嘛的

本项目是 [CatVodTVSpider](https://github.com/CatVodTVOfficial/CatVodTVSpider) 的一个派生/改版，本质是一个 **「爬虫（站点解析）开发与调试沙盒」**。

CatVod 系播放器（如 FongMi 的影视 App）本身只负责 UI 和播放，**真正抓取站点数据（首页 / 分类 / 详情 / 播放地址 / 搜索 / 直播）的逻辑，放在一个叫 `custom_spider.jar` 的插件包里**。本项目的作用就是：

1. 用 Java 编写各种站点的解析器（spider）；
2. 在手机 / 模拟器上用一个 Demo App 实时调试它们；
3. 打包出一个 `custom_spider.jar` 给播放器加载使用。

## 三、核心架构

### 1. 爬虫契约：`crawler/Spider.java`

所有站点解析器都继承这个抽象类，需要实现的关键方法：

| 方法 | 作用 |
| --- | --- |
| `init(Context, String extend)` | 初始化，`extend` 是配置里传入的扩展参数 |
| `homeContent(boolean filter)` | 首页分类 |
| `homeVideoContent()` | 首页推荐视频 |
| `categoryContent(tid, pg, filter, extend)` | 分类列表（分页 + 筛选） |
| `detailContent(List<String> ids)` | 详情 + 选集播放列表 |
| `searchContent(key, quick[, pg])` | 搜索 |
| `playerContent(flag, id, vipFlags)` | 解析真实播放地址 |
| `liveContent(url)` | 直播源 |
| `proxy(Map params)` | 本地代理（处理需要中转的请求） |

### 2. 配置：`json/config.json`

- `sites`：点播站点，`api` 字段用 `csp_类名` 指向对应 spider（如 `csp_Local`、`csp_Market`）。
- `lives`：直播源，同样用 `csp_类名`（如 `csp_XtreamCode`）。
- `spider`：指向 `custom_spider.jar`。
- `rules` / `ads` / `doh`：嗅探规则、广告过滤、DNS over HTTPS。

### 3. Demo 界面：`MainActivity.java`

界面是一排按钮，每个按钮对应 spider 的一个方法，点击后把返回的 JSON 直接显示在屏幕上，方便调试。

> 注意：当前调试目标是硬编码的 `spider = new PTT();`，想测哪个 spider 就改这一行。

## 四、如何开发（新增 / 修改一个站点）

1. 在 `app/src/main/java/com/github/catvod/spider/` 下新建类，继承 `Spider`，实现需要的方法。
   - 参考 `PTT.java`：用 `OkHttp.string(url, header)` 拉网页，用 `Jsoup` 解析 DOM，用 `Result.string(...)` 组装标准返回。
2. 在 `json/config.json` 的 `sites` 里新增一项，`"api": "csp_你的类名"`。
3. 在 `MainActivity` 中把 `spider = new PTT();` 改成你的类，即可在 Demo 里点按调试。

一个最小示例（伪代码）：

```java
public class MySite extends Spider {
    private final String url = "https://example.com/";

    @Override
    public String homeContent(boolean filter) {
        Document doc = Jsoup.parse(OkHttp.string(url, header()));
        List<Class> classes = new ArrayList<>();
        // 解析分类...
        return Result.string(classes);
    }

    @Override
    public String categoryContent(String tid, String pg, boolean filter, HashMap<String, String> ext) {
        // 解析列表 -> List<Vod>
        return Result.string(list);
    }

    @Override
    public String detailContent(List<String> ids) {
        Vod vod = new Vod();
        vod.setVodPlayFrom("线路1");
        vod.setVodPlayUrl("第1集$playId#第2集$playId2");
        return Result.string(vod);
    }

    @Override
    public String playerContent(String flag, String id, List<String> vipFlags) {
        return Result.get().url("真实播放地址").string();
    }
}
```

## 五、如何调试

1. 用 Android Studio 打开本项目，连接真机或启动模拟器。
2. 把 `MainActivity` 里的 `spider = new PTT();` 改成你要测试的 spider。
3. 直接 **Run `app`**，进入 App 后点击对应按钮：
   - `home` / `homeVideo` / `category` / `detail` / `player` / `search` / `live` / `proxy`。
4. 返回的 JSON 会显示在屏幕上，配合 **Logcat**（项目用了 `logger` 库）查看日志排错。

## 六、如何打包与使用（产出 custom_spider.jar）

构建流程（见 `build.bat` / `jar/genJar.bat` / `jar/checkJar.ps1`）：

1. `gradlew assembleRelease` → 打出 `app-release-unsigned.apk`。
2. 用 `apktool` 反编译 APK，抽出 `com/github/catvod/spider` 和 `js` 的 smali。
3. 重新打包成 `jar/custom_spider.jar`，并生成对应 `.md5`。
4. `checkJar.ps1` 做校验：核对 md5、检查是否混入了不该打包的依赖、Gson TypeToken 用法等。

最终把 `custom_spider.jar` 放进播放器的配置目录，配合 `config.json` 即可被加载使用。

### Windows 一键构建

```bat
build.bat
```

依次执行：编译 → 生成 jar → 校验。

## 七、macOS / Linux 环境说明

`build.bat` 和 `genJar.bat` 是 **Windows 批处理**，在 macOS 上不能直接运行。替代方案：

- 打 APK：`./gradlew assembleRelease`（跨平台可用）。
- 抽取 / 重打包：手动用 `apktool`（`brew install apktool`）执行 `genJar.bat` 里的等价步骤，或将其改写为 shell 脚本 `genJar.sh`。
- 校验：`checkJar.ps1` 是 PowerShell 脚本，macOS 安装 `pwsh`（`brew install --cask powershell`）后可跨平台执行。

## 九、播放器如何加载本项目的 jar（以 FongMi/TV 为例）

播放器源码仓库：<https://github.com/FongMi/TV>。
它内置一个 **catvod 模块**（基于 `CatVodTVOfficial/CatVodTVJarLoader`），负责在**不更新主程序**的前提下动态加载外部 `custom_spider.jar`。

### 1. 核心类

- `JarLoader` / `BaseLoader`：动态加载 JAR 的核心类。
- `VodConfig` / `LiveConfig`：解析点播 / 直播配置 JSON，触发 JAR 加载。
- `Spider`（与本项目 `crawler/Spider.java` 同契约）：所有解析器的统一接口。

### 2. 加载流程（`JarLoader` + `DexClassLoader`）

1. **确定 JAR 来源**：支持 HTTP URL、本地文件系统、或应用 `assets` 目录（本项目产出的 `custom_spider.jar` 走本地 / URL 路径）。
2. **完整性校验（可选）**：若配置里带 MD5，下载后先校验（对应本项目生成的 `custom_spider.jar.md5`）。
3. **类加载**：用 Android 的 `DexClassLoader` 从 JAR 里动态加载 `classes.dex` 的字节码。
4. **初始化 `Init` 类**：加载并调用 JAR 内的 `com.github.catvod.spider.Init`，传入 `Application` Context 完成扩展初始化（对应本项目 `spider/Init.java`）。
5. **保存 `Proxy` 类**：识别 `com.github.catvod.spider.Proxy`，供后续请求走本地代理（对应本项目 `spider/Proxy.java`）。
> JAR 加载前会被设为只读，类加载在受控环境中进行。

### 3. 站点如何引用具体的 Spider 类（`csp_` 约定）

这是本项目 `config.json` 里 `api: "csp_XXX"` 的由来：

- 反射类名的规则：`com.github.catvod.spider.[api 去掉 "csp_" 前缀]`
  - 例如 `api: "csp_Local"` → 加载 `com.github.catvod.spider.Local`
  - 例如 `api: "csp_XtreamCode"` → 加载 `com.github.catvod.spider.XtreamCode`
- Spider 实例被反射创建后，会用 `Application` Context + 扩展数据 `init()`；已加载实例以「JAR key + api 名」为键存入 `ConcurrentHashMap` 缓存，避免重复加载。
- 加载入口在 `JarLoader.getSpider(...)`。

### 4. config.json 的双重关联

播放器在解析配置时做两件事：

1. 从配置对象里取 `"spider"` 字段（即 `custom_spider.jar` 的地址），调用 `BaseLoader.get().parseJar(spider, true/false)` 加载 jar（`true`=点播，`false`=直播）。
2. 每个站点（site / live）则通过 `api` 字段的 `csp_` 前缀，告诉系统反射调用 jar 里的哪一个 Spider 类。

> 即：**`spider` 字段指向「jar 文件」，`api` 的 `csp_xxx` 指向「jar 里的具体类」**。

### 5. 与本项目结构的对应关系（为什么本项目能直接用）

| 播放器期望（约定） | 本项目实现 |
| --- | --- |
| `com.github.catvod.spider.Init` | `spider/Init.java` |
| `com.github.catvod.spider.Proxy` | `spider/Proxy.java` |
| `com.github.catvod.spider.<name>` | `spider/*.java`（PTT/Bili/AList/...） |
| `Spider` 抽象契约 | `crawler/Spider.java`（完全一致的方法签名） |
| `csp_<name>` 引用 | `json/config.json` 各站点 `api` 字段 |

因为命名空间、类名、契约都对齐，本项目打出的 `custom_spider.jar` 能被 FongMi/TV 等 CatVod 系播放器直接识别加载。

## 十、XBPQ 规则源（第三方规则引擎，可与本项目集成）

### 1. XBPQ 是什么

XBPQ 是 TVBox / CatVod 生态里一个**规则驱动型爬虫解析器**，本质是一个约 300KB 的 `xbpq.jar`（一个自定义的 csp spider，`type=3`），里面内置了一个**规则引擎**。

它把「写爬虫」简化成了「**写 JSON 配置文件（写源）**」：你不用为每个站点写 Java/JS 代码，只需用截取符号描述怎么从目标网页 / JSON 抽取分类、列表、详情、播放与搜索结果，引擎负责执行。

### 2. 三种写源方式对比

| 类型 | 实现方式 | 写源成本 | 例子 |
| --- | --- | --- | --- |
| **jar 源** | 每站一个 Java 类，编译进 spider jar | 高（需 Android 开发、反编译打包） | **本项目**（CatVodSpider） |
| **js 源** | 每站一个 JS 脚本，壳内嵌引擎执行 | 中 | jadehh/TVSpider |
| **XBPQ 规则源** | 只有一个 `xbpq.jar`（规则引擎），每站是一份 JSON 规则 | 低（填截取串即可） | XBPQ 源 |

### 3. 工作原理

- 配置里写 `"api": "csp_XBPQ"`，`"type": 3`，与本项目 `csp_Local` 是同一套识别机制。
- 在 `ext` 字段里写**规则**，引擎按四块截取整站数据：
  - **数组**：分类页抽影片卡片（标题 / 图片 / 链接）
  - **线路**：详情页抽多条播放线路
  - **播放**：每条线路抽分集与真实地址
  - **搜索**：搜索结果抽取
- 规则语法（截取符号）：
  - `&&` / `$$`：提取前后字符串之间的内容
  - `*`：通配符
  - `[不包含:关键词]` / `[替换:原>>新]` / `[排序:...]`：过滤、替换、排序
  - JSON 模式：直接写 `data.list[1].name` 等路径
  - `Base64()`：整体或局部解码
- **查错顺序**：先填 `分类url` → 没数据补 `数组` → 点不开详情补 `链接` → 没播放补 `播放数组` → 播不了补 `播放链接`；可开 `"调试":"1"`。

### 4. 与本项目的关系（可集成，非互斥）

- XBPQ 本身就是一个**第三方 spider jar**，加载方式与本项目完全一致（`DexClassLoader` + `csp_` 反射）。
- 可把 `xbpq.jar` 的 smali **「精缝」合并进本项目工程**，再走 `genJar.bat` 那套 apktool 抽 smali / 重打包流程，一起打出 `custom_spider.jar`。
- 集成后：新增站点只需写 JSON 规则，不必为每个站写 Java 类。

### 5. 你遇到的「XBPQ源」通常指什么

是别人维护的、基于 `xbpq.jar` 的**配置仓库**，例如 `boyss180/gao-xBPQ`（内含 `xBPQ.json` 纯 XBPQ 源，jar 与配置来自作者「菜妮丝」）。用户在 TVBox 里导入该 json 即可，jar 会随配置自动加载。这类仓库一般声明「复刻自用、切勿传播」。

> ⚠️ 「精缝」涉及反编译 / 修改第三方 jar 并重打包，操作时务必确认来源可信，避免引入安全隐患或导致编译失败。
